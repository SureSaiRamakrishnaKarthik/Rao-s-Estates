# Feature: Location CRUD

## Overview
The Location CRUD module allows administrators to define and manage geographic locations or regions (e.g., "Vijayawada", "Amaravati"). These locations are then assigned to projects and dynamically displayed on the public homepage.

## Architecture Reuse
Because the CRUD architecture is standardized, the Location feature heavily reuses components built for the Developer feature:
- `DataTable`
- `PageHeader`
- `ConfirmDialog`
- `EmptyState`

## Key Components

### 1. Data Schema (`src/schemas/location.schema.ts`)
- **Fields:**
  - `name`: String, min 2 chars.
  - `slug`: String, lowercase alphanumeric and hyphens only (auto-generated by `slugify`).
  - `description`: Optional text describing the region.

### 2. Service Layer (`src/services/location.service.ts`)
Interacts with the `locations` table in Supabase. Provides standard CRUD methods.

### 3. Server Actions (`src/app/admin/(dashboard)/locations/actions.ts`)
Provides server-side Zod validation and execution for creating, updating, and deleting locations. Ensures cache invalidation via `revalidatePath`.

### 4. Public Integration
Data managed through this CRUD interface is directly queried by `HomeService` and rendered in the `FeaturedLocations` component on the landing page, replacing static placeholder data.
